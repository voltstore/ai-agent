# اتقان (Itqan) — متجر قطع الكمبيوتر والتجميعات الاحترافية

نموذج أولي عالي الدقة (High-Fidelity Prototype) — HTML/CSS/JS خام بدون أي frameworks أو build tools.

**v3**: هوية **أبيض + برتقالي + بني** — فاتح دافئ افتراضي + ثيم داكن «إسبريسو»،
**واجهة ثنائية اللغة (عربي RTL / إنجليزي LTR)**، أنيميشن سكرول، ولوحة تحكم كاملة
(إحصائيات + بحث شامل + إعدادات المتجر).

## التشغيل

افتح `index.html` مباشرة في المتصفح. لا يحتاج أي تثبيت.

- **الثيم**: الفاتح (أبيض دافئ) افتراضي، والداكن «إسبريسو» من زر الشمس/القمر.
- **اللغة**: العربية افتراضية، والتبديل من زر `English/عربي` في الهيدر (يُحفظ في localStorage).
  كل النصوص في `i18n.js` — أضف مفتاحًا هناك ثم استخدم `data-i18n` أو `L()`.
- **رقم الواتساب**: يُدار من لوحة التحكم `admin.html` (قسم إعدادات المتجر) — يُحفظ في `settings.js`.

## الإعداد المطلوب قبل الإطلاق

1. **رقم الواتساب**: في أعلى `app.js` عدّل الثابت `WHATSAPP_PHONE`
   (صيغة دولية بدون `+`، مثال: `966512345678`).
2. **مفتاح Gemini**: لا يحتاج إعدادًا منك — كل مستخدم يُدخل مفتاحه عند أول
   فتح للمساعد الذكي، ويُحفظ في `localStorage` على جهازه فقط.
   مفتاح مجاني من: https://aistudio.google.com/app/apikey

## بنية المشروع

| الملف | المسؤولية |
|-------|-----------|
| `index.html` | الهيكل (Hero، مميزات، البنّاء، خطوات، السلة، الفوتر، المحادثة) |
| `admin.html` + `admin.js` | **لوحة إدارة المخزون** (محلية لصاحب المتجر — لا تُنشر) |
| `style.css` | نظام التصميم — ثيم داكن (افتراضي) + فاتح عبر CSS variables |
| `i18n.js` | **قواميس الترجمة** عربي/إنجليزي — كل نصوص الواجهة ورسائل التوافق |
| `inventory.js` | **المخزون الخام** — يولَّد آليًا من لوحة الإدارة (لا تعدّله يدويًا) |
| `data.js` | **طبقة البيانات** — يقرأ inventory.js ويوفر `getParts()` |
| `compat.js` | **محرك التوافق** — نقي وقابل للاختبار، يرجع أكواد أسباب مهيكلة (بدون نصوص) |
| `app.js` | واجهة البنّاء، السلة، تصدير واتساب، الثيم |
| `chat.js` | مساعد Gemini الذكي (REST مباشر، موديل `gemini-2.0-flash`) |
| `images/` | صور SVG مؤقتة لكل فئة — استبدلها بصور حقيقية بنفس المسارات |
| `tests/compat.test.js` | اختبارات محرك التوافق — شغّلها بـ `node tests/compat.test.js` |
| `_preview/` | لقطات شاشة للمعاينة — يمكن حذف المجلد بأمان |

## إضافة قطع جديدة (لوحة الإدارة)

1. افتح `admin.html` في Chrome أو Edge.
2. اضغط **«اربط مجلد المشروع»** واختر مجلد `ittcan store` (مرة واحدة لكل جلسة).
3. اختر الفئة → **«＋ إضافة قطعة»** → عبّئ الاسم والسعر والمواصفات وارفق الصورة.
4. الحفظ تلقائي: تُكتب القطعة في `inventory.js` وتُنسخ الصورة إلى `images/<id>.<ext>`.
5. بدون ربط المجلد: استخدم زر «تنزيل inventory.js» واستبدل الملف يدويًا.

## قواعد التوافق المطبّقة

1. سوكِت المعالج ↔ سوكِت اللوحة الأم (AM4 / AM5 / LGA1700)
2. نوع الذاكرة DDR4/DDR5 ↔ دعم اللوحة الأم
3. قدرة مزود الطاقة ≥ (استهلاك المعالج + كرت الشاشة + 100 واط) × 1.2 هامش أمان
4. الصندوق يدعم مقاس اللوحة (ATX / mATX)
5. المبرد يغطي حرارة المعالج (TDP)

## المرحلة الثانية (Firebase)

استبدل جسم `getParts()` في `data.js` بقراءة من Firebase Realtime Database —
لا يتغير أي شيء آخر في التطبيق:

```js
async function getParts() {
  const snapshot = await firebase.database().ref('parts').get();
  return snapshot.val();
}
```
